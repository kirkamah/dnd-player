# Формат `.dndsession` — спецификация v1.1

`.dndsession` — это **zip-архив** с записью D&D-сессии: мультитрек-звук по игрокам,
таймлайн «кто когда говорил», арты персонажей и слои сцены. Производитель формата —
бот-рекордер (Проект 1). Потребители — плеер (Проект 2) и редактор (Проект 3).

Этот файл — **источник правды**. Любое изменение формата = новая `formatVersion`.

---

## Структура архива

```
session.dndsession  (zip)
├── manifest.json               # обязателен
├── audio/
│   ├── player-<userId>.wav     # по одному WAV на участника
│   └── ...
├── art/                        # опционально (если арты назначены через /assign)
│   ├── <characterId>-idle.png
│   ├── <characterId>-speaking.png
│   ├── frame.png
│   ├── bricks.png
│   └── background-<id>.png
├── music/                      # v1.1, опционально: добавленная в редакторе музыка
│   └── <имя файла>
└── reference.mp4               # опционально, превью-рендер (бот его не пишет)
```

## Аудио — жёсткие гарантии

| Параметр       | Значение                                   |
|----------------|--------------------------------------------|
| Контейнер      | WAV (RIFF), канонический 44-байтный заголовок |
| Кодек          | PCM s16le                                  |
| Sample rate    | **48000 Гц**                               |
| Каналы         | **1 (моно)** — Discord отдаёт голос каждого участника фактически в моно; стерео удваивало бы размер без пользы |

**Инварианты, на которые могут полагаться плеер и редактор:**

1. **Все дорожки одной длины с точностью до сэмпла.** Число сэмплов каждой дорожки
   равно `round(durationMs * 48)`.
2. Все дорожки выровнены по общему нулю времени `t0` (момент старта записи).
   Паузы и время до подключения участника заполнены цифровой тишиной (нулями).
3. `durationMs` в манифесте — длительность сессии; `byteLength(data) == round(durationMs*48) * 2`
   для каждого WAV.

## manifest.json

```jsonc
{
  "formatVersion": "1.0",
  "sessionId": "uuid v4",
  "recordedAt": "2026-06-06T15:00:00.000Z",   // ISO 8601, UTC
  "durationMs": 5400000,
  "sampleRate": 48000,
  "channels": 1,

  // Мастер. null, если никому не назначен слот "master".
  "master": {
    "userId": "discord snowflake (string)",
    "displayName": "Имя в Discord на момент записи",
    "characterId": "master",                  // kebab-case id, уникален в бандле
    "characterName": "Мастер",
    "slot": "master",
    "audioFile": "audio/player-<userId>.wav",
    "art": {                                  // опционально; ключи тоже опциональны
      "idle": "art/master-idle.png",
      "speaking": "art/master-speaking.png"
    }
  },

  // Игроки (все записанные участники, кроме мастера).
  "players": [
    {
      "userId": "...",
      "displayName": "...",
      "characterId": "char-1",
      "characterName": "Гэндальф",
      "slot": 0,                              // 0..5 — позиция в одной из 6 рамок;
                                              // null — участник записан, но слот не назначен
      "audioFile": "audio/player-<userId>.wav",
      "art": { "idle": "art/char-1-idle.png", "speaking": "art/char-1-speaking.png" }
    }
  ],

  // Таймлайн речи: драйвит смену арта idle <-> speaking.
  // Отсортирован по startMs. Интервалы одного userId не пересекаются.
  // 0 <= startMs < endMs <= durationMs.
  "speakingEvents": [
    { "userId": "...", "startMs": 1200, "endMs": 2600 }
  ],

  // Слои сцены (снизу вверх: background -> bricks -> frame -> портреты).
  // Все ключи опциональны: бот кладёт их, только если ассеты заданы (ASSETS_DIR).
  "layers": {
    "background": "art/background-tavern.png",
    "bricks": "art/bricks.png",
    "frame": "art/frame.png"
  },

  // Ключевые точки сцены. Бот пишет дефолт [{ "tMs": 0, "bricksOpacity": 1.0 }];
  // полноценно редактируется в Проекте 3.
  "sceneCues": [
    { "tMs": 0, "bricksOpacity": 1.0 },
    { "tMs": 30000, "bricksOpacity": 0.0, "background": "art/background-cave.png" }
  ],

  // --- v1.1, опционально: правки редактора (бот это поле не пишет) ---
  "edit": {
    // громкость/мьют дорожек участников (по умолчанию gain 1.0, muted false)
    "tracks": {
      "<userId>": { "gain": 0.8, "muted": false }
    },
    // добавленная музыка: файл лежит в music/ внутри архива
    "music": [
      { "file": "music/tavern-theme.mp3", "startMs": 0, "gain": 0.5 }
    ],
    // свои картинки поверх сцены (слой overlay, поверх портретов)
    "overlays": [
      {
        "image": "art/overlay-map.png",
        "startMs": 10000,
        "endMs": 25000,
        "x": 480, "y": 120, "w": 960, "h": 720,   // в координатах сцены 1920x1080
        "opacity": 1.0
      }
    ]
  }
}
```

### Правила

- Все пути внутри манифеста — **относительные к корню архива**, разделитель `/`.
- `userId` — всегда строка (Discord snowflake не влезает в double).
- `slot` уникален среди участников: каждое значение `0..5` и `"master"` встречается
  не более одного раза. `null` допускается у нескольких.
- `characterId` уникален; для участников без `/assign` бот генерирует `user-<userId>`,
  `characterName` = `displayName`.
- Участник без речи всё равно попадает в манифест с дорожкой из тишины,
  если он был в канале (дорожка создаётся при первом звуке; молчавшие всю
  сессию участники могут отсутствовать — потребитель не должен этого требовать).
- Неизвестные потребителю поля игнорируются (forward-совместимость).
  Редактор (Проект 3) расширяет манифест новыми полями, не ломая `formatVersion`.

## Версионирование

- `formatVersion` — semver-подобная строка `major.minor`.
- Минорные версии добавляют только опциональные поля.
- Потребитель обязан отказаться открывать бандл с неизвестным major.

История:
- **1.0** — базовый формат (бот-рекордер).
- **1.1** — опциональный блок `edit` (громкость/мьют дорожек, музыка `music/`,
  overlay-картинки) и папка `music/`. Бандлы 1.0 валидны как 1.1; редактор,
  сохраняя правки, пишет `formatVersion: "1.1"`.

## Валидация

В репозитории бота есть скрипт `npm run validate -- path/to/file.dndsession`,
который проверяет все инварианты выше. Любой производитель/редактор бандла
должен проходить эту проверку.
