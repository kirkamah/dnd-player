# Формат `.dndsession` — спецификация v1.4

`.dndsession` — это **zip-архив** с записью D&D-сессии: мультитрек-звук по игрокам,
таймлайн «кто когда говорил», арты персонажей и слои сцены. Производитель формата —
бот-рекордер (Проект 1). Потребители — плеер (Проект 2) и редактор (Проект 3).

Этот файл — **источник правды**. Любое изменение формата = новая `formatVersion`.

---

## Структура архива

```
session.dndsession  (zip)
├── manifest.json               # обязателен
├── audio/
│   ├── player-<userId>.wav     # по одному WAV на участника
│   └── ...
├── art/                        # опционально (если арты назначены через /assign)
│   ├── <characterId>-idle.png
│   ├── <characterId>-speaking.png
│   ├── frame.png
│   ├── bricks.png
│   └── background-<id>.png
├── music/                      # v1.1, опционально: добавленная в редакторе музыка
│   └── <имя файла>
└── reference.mp4               # опционально, превью-рендер (бот его не пишет)
```

## Аудио — жёсткие гарантии

| Параметр       | Значение                                   |
|----------------|--------------------------------------------|
| Контейнер      | WAV (RIFF), канонический 44-байтный заголовок |
| Кодек          | PCM s16le                                  |
| Sample rate    | **48000 Гц**                               |
| Каналы         | **1 (моно)** — Discord отдаёт голос каждого участника фактически в моно; стерео удваивало бы размер без пользы |

**Инварианты, на которые могут полагаться плеер и редактор:**

1. **Все дорожки одной длины с точностью до сэмпла.** Число сэмплов каждой дорожки
   равно `round(durationMs * 48)`.
2. Все дорожки выровнены по общему нулю времени `t0` (момент старта записи).
   Паузы и время до подключения участника заполнены цифровой тишиной (нулями).
3. `durationMs` в манифесте — длительность сессии; `byteLength(data) == round(durationMs*48) * 2`
   для каждого WAV.

## manifest.json

```jsonc
{
  "formatVersion": "1.0",
  "sessionId": "uuid v4",
  "recordedAt": "2026-06-06T15:00:00.000Z",   // ISO 8601, UTC
  "durationMs": 5400000,
  "sampleRate": 48000,
  "channels": 1,

  // Мастер. null, если никому не назначен слот "master".
  "master": {
    "userId": "discord snowflake (string)",
    "displayName": "Имя в Discord на момент записи",
    "characterId": "master",                  // kebab-case id, уникален в бандле
    "characterName": "Мастер",
    "slot": "master",
    "audioFile": "audio/player-<userId>.wav",
    "art": {                                  // опционально; ключи тоже опциональны
      "idle": "art/master-idle.png",
      "speaking": "art/master-speaking.png"
    }
  },

  // Игроки (все записанные участники, кроме мастера).
  "players": [
    {
      "userId": "...",
      "displayName": "...",
      "characterId": "char-1",
      "characterName": "Гэндальф",
      "slot": 0,                              // 0..5 — позиция в одной из 6 рамок;
                                              // null — участник записан, но слот не назначен
      "audioFile": "audio/player-<userId>.wav",
      "art": { "idle": "art/char-1-idle.png", "speaking": "art/char-1-speaking.png" }
    }
  ],

  // Таймлайн речи: драйвит смену арта idle <-> speaking.
  // Отсортирован по startMs. Интервалы одного userId не пересекаются.
  // 0 <= startMs < endMs <= durationMs.
  //
  // v1.2: реплика — это КЛИП. srcStartMs (опционально) — откуда в исходной
  // дорожке берётся звук этой реплики; длина источника равна endMs-startMs.
  // Если srcStartMs отсутствует, звук берётся с позиции startMs (как записано).
  // Потребитель, проигрывающий клипами: звук = сумма клипов, тишина между ними.
  // v1.3: gain (опционально, по умолчанию 1.0) — громкость конкретной реплики,
  // умножается на громкость дорожки участника (edit.tracks[userId].gain).
  "speakingEvents": [
    { "userId": "...", "startMs": 1200, "endMs": 2600, "srcStartMs": 1200, "gain": 1.0 }
  ],

  // Слои сцены. Порядок отрисовки СНИЗУ ВВЕРХ (v1.3):
  //   bricks (переключаемый фон-стена, прозрачность по sceneCues)
  //   -> background (рамка фона; может иметь прозрачное окно, сквозь которое
  //      виден bricks) -> портреты -> overlay-картинки -> frame (рамка
  //      портретов, НАИВЫСШИЙ слой).
  // Все ключи опциональны: бот кладёт их, только если ассеты заданы (ASSETS_DIR).
  "layers": {
    "background": "art/background-tavern.png",
    "bricks": "art/bricks.png",
    "frame": "art/frame.png"
  },

  // Ключевые точки сцены. Бот пишет дефолт [{ "tMs": 0, "bricksOpacity": 1.0 }];
  // полноценно редактируется в Проекте 3.
  // v1.4: fadeMs — плавный переход bricksOpacity к значению этого ключа
  // (линейно от предыдущего значения, начиная с tMs). 0/нет — мгновенно.
  "sceneCues": [
    { "tMs": 0, "bricksOpacity": 1.0 },
    { "tMs": 30000, "bricksOpacity": 0.0, "fadeMs": 2000,
      "background": "art/background-cave.png" }
  ],

  // --- v1.1, опционально: правки редактора (бот это поле не пишет) ---
  "edit": {
    // громкость/мьют дорожек участников (по умолчанию gain 1.0, muted false)
    "tracks": {
      "<userId>": { "gain": 0.8, "muted": false }
    },
    // добавленная музыка: файл лежит в music/ внутри архива.
    // v1.2: endMs — конец окна на таймлайне (по умолчанию старт + длина файла),
    // srcStartMs — смещение внутри файла (по умолчанию 0). Вместе дают обрезку.
    "music": [
      { "file": "music/tavern-theme.mp3", "startMs": 0, "gain": 0.5,
        "endMs": 95000, "srcStartMs": 0 }
    ],
    // свои картинки на сцене.
    // v1.4: layer — на каком плане рисуется картинка:
    //   "back"    — ЗА переключаемым фоном (ниже кирпичей, самый низ)
    //   "scene"   — над рамкой фона, под портретами
    //   "default" — над портретами, под рамкой портретов (как раньше)
    //   "front"   — ПОВЕРХ рамки портретов (самый верх)
    // v1.4: fadeInMs/fadeOutMs — плавное появление/исчезание (рампа
    // прозрачности у краёв интервала). Порядок в массиве = порядок отрисовки
    // внутри одного плана (последний — выше).
    "overlays": [
      {
        "image": "art/overlay-map.png",
        "startMs": 10000,
        "endMs": 25000,
        "x": 480, "y": 120, "w": 960, "h": 720,   // в координатах сцены 1920x1080
        "opacity": 1.0,
        "layer": "default",
        "fadeInMs": 500,
        "fadeOutMs": 1000
      }
    ],

    // v1.2: позиции/размеры портретов, заданные в редакторе (поверх дефолтного
    // лейаута потребителя). Ключ — userId. hidden — портрет не рисуется.
    // v1.3: glow/glowColor/glowSize — свечение вокруг говорящего (по умолчанию
    // включено, цвет = style.speakingColor, размер 28); radius — персональное
    // скругление углов портрета (по умолчанию style.radius, клампится до круга).
    "layout": {
      "<userId>": {
        "x": 40, "y": 664, "w": 340, "h": 340, "hidden": false,
        "glow": true, "glowColor": "#2FA37C", "glowSize": 28, "radius": 170
      }
    },

    // v1.4: «таблички» — картинки с именем персонажа (обычно вытянутые
    // горизонтальные). Рисуются ВЫШЕ рамки портретов (под front-картинками).
    // Если у участника есть табличка, текстовая подпись под портретом
    // не рисуется. Ключ — userId.
    "plates": {
      "<userId>": { "image": "art/plate-gendalf.png", "x": 30, "y": 950, "w": 360, "h": 80, "hidden": false }
    },

    // v1.3: положение/размер слоя frame (рамки портретов). Если нет —
    // рамка растягивается на всю сцену (поведение старых бандлов).
    // locked: true — рамка зафиксирована, на превью редактора не таскается.
    "frameBox": { "x": 40, "y": 664, "w": 360, "h": 360, "locked": true },

    // v1.2: стиль портретов (обводка)
    "style": {
      "borderColor": "#39434f",     // обводка молчащего
      "speakingColor": "#2FA37C",   // обводка и свечение говорящего
      "borderWidth": 2,             // толщина обводки молчащего, px сцены
      "radius": 14                  // скругление углов
    }
  }
}
```

### Правила

- Все пути внутри манифеста — **относительные к корню архива**, разделитель `/`.
- `userId` — всегда строка (Discord snowflake не влезает в double).
- `slot` уникален среди участников: каждое значение `0..5` и `"master"` встречается
  не более одного раза. `null` допускается у нескольких.
- `characterId` уникален; для участников без `/assign` бот генерирует `user-<userId>`,
  `characterName` = `displayName`.
- Участник без речи всё равно попадает в манифест с дорожкой из тишины,
  если он был в канале (дорожка создаётся при первом звуке; молчавшие всю
  сессию участники могут отсутствовать — потребитель не должен этого требовать).
- Неизвестные потребителю поля игнорируются (forward-совместимость).
  Редактор (Проект 3) расширяет манифест новыми полями, не ломая `formatVersion`.

## Версионирование

- `formatVersion` — semver-подобная строка `major.minor`.
- Минорные версии добавляют только опциональные поля.
- Потребитель обязан отказаться открывать бандл с неизвестным major.

История:
- **1.0** — базовый формат (бот-рекордер).
- **1.1** — опциональный блок `edit` (громкость/мьют дорожек, музыка `music/`,
  overlay-картинки) и папка `music/`.
- **1.2** — клипы: `speakingEvents[].srcStartMs` (звук реплики едет за блоком),
  окно музыки `endMs`/`srcStartMs`, `edit.layout` (позиции/размеры/скрытие
  портретов), `edit.style` (обводка портретов).
- **1.3** — `speakingEvents[].gain` (громкость отдельной реплики),
  `edit.layout[].glow/glowColor/glowSize` (свечение говорящего на портрет),
  `edit.frameBox` (положение/размер/блокировка рамки портретов),
  новый порядок слоёв (bricks → background → портреты → overlays → frame),
  `style.radius` клампится до круга (`min(w,h)/2`).
- **1.4** — `overlays[].layer` (план картинки: back/scene/default/front),
  `overlays[].fadeInMs/fadeOutMs` (плавное появление/исчезание),
  `sceneCues[].fadeMs` (плавный переход прозрачности кирпичей),
  `edit.plates` (таблички с именами поверх рамки портретов).
  Полный порядок отрисовки: bricks → overlays(back) → background →
  overlays(scene) → портреты → overlays(default) → frame → plates →
  overlays(front).
  Бандлы 1.0–1.3 валидны как 1.4; редактор пишет `formatVersion: "1.4"`.

## Валидация

В репозитории бота есть скрипт `npm run validate -- path/to/file.dndsession`,
который проверяет все инварианты выше. Любой производитель/редактор бандла
должен проходить эту проверку.
